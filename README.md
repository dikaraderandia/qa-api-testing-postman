# QA API Testing Project (Postman)

## Overview
This project demonstrates API testing using Postman, covering authentication scenarios for an e-commerce API.

## Scope
- Login API testing
- Positive and negative test scenarios
- Response validation

## Tools
- Postman

## Test Scenarios
- Valid login
- Invalid password
- Missing fields
- Null values
- Invalid request method

## Validation
- Status code verification
- Response body validation
- Error handling

## How to Run
1. Import collection into Postman
2. Run requests individually or via Collection Runner

## Notes
This project focuses on real API behavior validation and negative testing scenarios.
